This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Marketing site for "οι Financial Greeks" (Greek finance content brand). Greek-language single-page app with a handful of legal subpages. Built as a **client-only React SPA** with Vite, deployed as plain static files (`html` / `js` / `css`) on a regular Apache shared host. Originally a TanStack Start SSR app on Cloudflare Workers — that infrastructure was removed; see "Migration history" at the bottom if you need context.

## Commands

- `npm run dev` — Vite dev server.
- `npm run build` — production build → `dist/` (static files ready to upload).
- `npm run preview` — preview the production build locally.
- `npm run lint` — ESLint (TypeScript + react-hooks + prettier).
- `npm run format` — Prettier write across the repo.

There is no test framework configured and no separate typecheck script (TS errors surface during `vite build`).

## Deployment

Static deploy to cPanel / shared hosting. After `npm run build`, upload the **contents** of `dist/` (not the folder itself) into the host's `public_html/` over FTP/SFTP. The bundle is:

- `index.html` — SPA shell
- `assets/*.js`, `assets/*.css` — content-hashed bundles
- `.htaccess` — Apache rewrite rules (copied from `public/.htaccess`)
- `favicon.ico`

**`.htaccess` is mandatory.** It rewrites every non-file path to `/index.html` so direct visits to `/privacy`, `/terms`, `/cookies`, `/corporate` actually load the app instead of 404'ing. If the host disables `.htaccess` overrides we'd have to fall back to hash routing (`/#/privacy`). The same file also sets long cache headers on hashed assets and `no-cache` on `index.html`.

## Architecture

**Stack:** Vite + React 19 + TanStack Router (file-based, client-only) + TanStack Query, Tailwind v4, shadcn/ui (new-york style, slate base). No SSR, no server functions, no runtime backend — everything ships as static assets.

**Path alias:** `@/*` → `src/*` (via `tsconfig.json` + `vite-tsconfig-paths`).

### Entry points

- `index.html` (project root) — Vite's SPA shell. Holds the static `<title>`, OG/Twitter meta, Google Fonts (Sora + Inter, Greek subsets), and `<div id="root"></div>`. Initial document title comes from here; per-route titles are set client-side.
- `src/main.tsx` — client bootstrap. Creates the `QueryClient` and the TanStack Router, wraps `<RouterProvider />` in `<QueryClientProvider />`, mounts into `#root`. Declares the router type via `declare module "@tanstack/react-router"` for end-to-end type safety.

### Routing

File-based via TanStack Router in `src/routes/`. `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin/vite` (configured in `vite.config.ts` with `autoCodeSplitting: true`) — **never edit by hand**. Routes: `/` (`index.tsx`), `/corporate`, `/privacy`, `/terms`, `/cookies`. `__root.tsx` renders `<Outlet />` + `<CookieBanner />` plus the shared 404 / error UIs.

### Per-page meta in SPA mode

There's no SSR head manager. Each route component sets its own `document.title` via a small `useEffect(() => { document.title = TITLE; }, [])`. If a route needs to change description/OG tags too, add a similar effect that mutates the relevant `<meta>` element — don't try to reintroduce the TanStack Router `head()` / `<HeadContent />` API, it relies on infrastructure that was removed.

**SEO caveat:** because rendering is client-only, search-engine and social-preview crawlers that don't execute JS will only see the static meta in `index.html`. Google handles JS rendering, but Facebook/Twitter/LinkedIn previews effectively use the homepage meta regardless of the URL. If per-page social previews become important, the migration to a prerender setup (one HTML file per route at build time) is the right next step.

### UI components

shadcn/ui in `src/components/ui/` (configured via `components.json`: `style: "new-york"`, `baseColor: "slate"`, icon library `lucide`). Site-specific components in `src/components/site/`: `Chrome.tsx` (header + footer), `CookieBanner.tsx`, `LegalLayout.tsx`, `VideoThumb.tsx`.

### Styling

Tailwind v4 with CSS-first config in `src/styles.css` (`@import "tailwindcss"`, `@theme inline { ... }`). Custom design tokens: `--coral`, `--mint`, `--ink`, `--surface`, plus gradient utilities (`bg-grad-coral`, etc.) and `shadow-glow`. Dark creator palette in oklch. Greek + Latin font subsets are loaded from Google Fonts in `index.html`.

### Cookie consent

`CookieBanner` persists the choice to `localStorage` under the key `fg-cookie-consent` (`accepted` | `rejected`). No analytics scripts are loaded — the banner is informational and a placeholder for future tracking.

### YouTube content

Currently **static thumbnails only** (see `heroVideos` / `recentVideos` arrays in `src/routes/index.tsx`). The previous live YouTube RSS integration was removed because (a) the SSR server function went away with the stack migration, and (b) YouTube's `feeds/videos.xml` endpoint can't be called from a browser due to CORS. If a live feed is needed again, it must run through a small serverless proxy (Cloudflare Worker / Netlify Function / etc.) that fronts the RSS endpoint and adds permissive CORS headers.

## Conventions

- Greek copy throughout user-facing UI. Keep that language unless the user explicitly asks for translation.
- Brand-owned external links live as module-level constants at the top of components (e.g. `YT`, `IG`, `LI`, `MAIL` in `Chrome.tsx`). Keep them centralized — they recur across pages.
- Recharts, react-hook-form, zod, and many Radix primitives are already installed but the current site does not use most of them. Don't assume they're wired up.

## Migration history (for context only)

This project started life from the Lovable `tanstack_start_ts_2026-05-06` template — TanStack Start (SSR) on Cloudflare Workers, with a `getLatestVideos` server function pulling the YouTube RSS feed. It was converted to a client-only SPA so it could be deployed on plain Apache shared hosting via FTP. The following were removed in that migration: `src/server.ts`, `src/start.ts`, `src/lib/error-capture.ts`, `src/lib/error-page.ts`, `src/lib/youtube.functions.ts`, `src/router.tsx`, `wrangler.jsonc`, and the `@tanstack/react-start`, `@cloudflare/vite-plugin`, `@lovable.dev/vite-tanstack-config` dependencies. Don't reintroduce them without re-planning the deploy target.
